# data-seiton2-soudan

相談者データの整理（重複削除と最新化）を自動化・標準化するための指示セットです。

## このスキルを使用する時

- 整理番号が同じ人を同一人物として扱いたい。
- 複数ある相談記録のうち、「相談が成立している」かつ「最も遅い日程（最新）」のものだけを残したい。
- 分析用にデータを1名1行に集約したい。

## このスキルを使用しない時

- 経時的な相談変化を分析するために全履歴が必要な場合。
- 整理番号が欠落しているデータのクレンジング。

## ワークフロー

1. **データの読み込みとグルーピング**
   - 対象の相談データファイルを読み込む。
   - 「整理番号」列をキーとしてデータをグループ化する。

2. **最新データの特定**
   - 各グループ内で「相談が成立している」行をフィルタリングする。
   - フィルタリングされた行の中から、「相談日」または「日程」列が最も遅い（最新の）行を特定する。

3. **重複の削除と出力**
   - 特定した最新の1行のみを保持し、それ以外の重複する行（古い履歴や相談未成立の行）を削除する。
   - 整頓された結果を新しいファイル、または指定された形式で出力する。

## 実装上の注意点

- **日付形式の統一**: 日程の比較を行う前に、日付列が正しく日時型として解釈されているか確認すること。
- **相談成立の定義**: 「相談成立」がどの列のどの値（例：ステータス列が"完了"など）を指すか、事前に確認すること。
- **データのバックアップ**: 削除を伴う処理のため、元のデータを直接上書きせず、別名で保存することを推奨。

## ヒアリング項目

処理を開始する前に以下を確認してください：
- 重複排除の対象となるファイル名
- 「整理番号」に対応する正確な列名
- 「相談が成立している」ことを判断する列名と具体的な値
- 「日程」を表す列名
